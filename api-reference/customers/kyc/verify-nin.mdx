---
title: 'Verify NIN'
api: 'POST https://api.fossapay.com/v1/customers/kyc/nin'
description: 'Verify NIN (National Identification Number) for customer'
---

## Headers

<ParamField header="x-api-key" type="string" required>
  API Key for authentication
</ParamField>

## Request Body

<ParamField body="customerId" type="string" required>
  Customer ID
</ParamField>

<ParamField body="nin" type="string" required>
  National Identification Number
</ParamField>

<ParamField body="firstName" type="string" required>
  Customer's first name
</ParamField>

<ParamField body="lastName" type="string" required>
  Customer's last name
</ParamField>

<ParamField body="dob" type="string" required>
  Date of birth (ISO date string)
</ParamField>

## Request Example

```bash
curl -X POST https://api.fossapay.com/v1/customers/kyc/nin \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "customerId": "cust_123456789",
    "nin": "12345678901",
    "firstName": "John",
    "lastName": "Doe",
    "dob": "1990-01-15"
  }'
```

```javascript
const response = await fetch('https://api.fossapay.com/v1/customers/kyc/nin', {
  method: 'POST',
  headers: {
    'x-api-key': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    customerId: 'cust_123456789',
    nin: '12345678901',
    firstName: 'John',
    lastName: 'Doe',
    dob: '1990-01-15'
  })
});
```

## Notes

- The NIN verification process may take several seconds to complete
- Ensure all customer details match exactly with the NIN records
- This endpoint triggers KYC level upgrade for the customer upon successful verification
